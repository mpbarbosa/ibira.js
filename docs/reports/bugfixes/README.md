# `docs/reports/bugfixes`

Post-mortem and fix-log reports for resolved bugs and regressions.

## What belongs here

After fixing a non-trivial bug, add a short report here explaining:

- What the bug was and how it was observed
- Root cause
- Fix applied and why
- How to verify it stays fixed (test reference or manual steps)

Name files descriptively: `YYYY-MM-fix-short-description.md`.
