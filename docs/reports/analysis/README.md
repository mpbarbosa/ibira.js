# `docs/reports/analysis`

Architectural validation and code quality analysis reports.

## Contents

| File | Description |
| ---- | ----------- |
| `ARCHITECTURAL_VALIDATION_REPORT.md` | Validates the dual-layer architecture against design goals |
| `CODE_QUALITY_ASSESSMENT.md` | Static analysis and quality metrics snapshot |
| `DOCUMENTATION_REPORT.md` | Coverage and completeness assessment of repository documentation |

## What belongs here

Point-in-time analysis reports. Add new reports with a date prefix (e.g. `2026-06-ANALYSIS.md`) to preserve history. Do not overwrite existing reports.
