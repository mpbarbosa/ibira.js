# AI Workflow Automation

This directory contains tooling artifacts generated by the AI workflow automation system used for code review, documentation analysis, and incremental development of ibira.js.

## Structure

```
.ai_workflow/
├── backlog/             # Pending tasks and feature backlog items
├── commit_history.json  # Automated commit metadata log
├── logs/                # Per-workflow-run prompt/response logs
│   └── workflow_YYYYMMDD_HHMMSS/
│       └── prompts/
│           └── step_NN/ # Individual agent outputs (documentation, devops, test, etc.)
├── metrics/             # Collected quality and coverage metrics
└── prompts/             # Reusable prompt templates for AI agents
```

## Purpose

These artifacts are **not library code** — they are internal tooling that supports the development process:

- **logs/** — Full transparency trail of AI-assisted analysis (code quality, dependency, test coverage, architecture reviews)
- **backlog/** — Incremental improvement items discovered during automated review passes
- **metrics/** — Snapshot quality metrics used to track progress across workflow runs

## Note

This directory is intentionally excluded from npm publication (not in `files` in `package.json`) and does not affect library consumers.
